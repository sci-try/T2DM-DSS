Injectable therapy decision algorithm

This page describes the current decision logic used by the CDS prototype for adults with type 2 diabetes (T2D). The tool is designed as a practical, reimbursement-oriented support algorithm for injectable therapy selection, intensification, and simplification.

Algorithm version: v2 Scope: first injectable, intensification, simplification Countries: TR, LB, IQ Engine: py/engine.py

Textual description of the algorithm

1) Country selection

The current engine supports only Turkey (TR), Lebanon (LB), and Iraq (IQ). In this version, FRC is treated as available by default in all three countries.

For Turkey, if the final recommendation is FRC and BMI < 35 kg/m², the engine adds a reimbursement note indicating that treatment may be partially or fully out-of-pocket depending on local access conditions.

2) Severe hyperglycaemia override

If catabolic symptoms are present or HbA1c ≥ 10%, the algorithm immediately recommends start / intensify insulin.

This is an override step: severe hyperglycaemia takes priority over all other routing branches.

3) Intensification on FRC

If the patient is already on FRC and glycaemic target remains unmet, the algorithm recommends adding rapid-acting insulin.

If the patient is already on FRC + rapid-acting insulin and still uncontrolled, the next step is intensification to basal-bolus or premixed insulin.

A note is added that adding rapid-acting insulin to FRC may be off-label, depending on local label and market.

4) Simplification from complex insulin regimens

If the patient is on basal-bolus or premixed insulin and has recurrent hypoglycaemia, the algorithm recommends: consider switch to FRC for simplification.

In this version, regimen complexity is treated as intrinsically relevant to basal-bolus and premix, so a separate complexity checkbox is no longer used.

5) Intensification on basal insulin

If the patient is on basal insulin and either:

  • HbA1c remains above target, or
  • Postprandial glucose is uncontrolled,

then the algorithm recommends: switch basal insulin to FRC.

This branch captures both overall inadequate control and the common scenario of acceptable fasting control with persistent postprandial hyperglycaemia.

6) First injectable selection

If the patient is not already routed into a more advanced intensification or simplification branch, the engine uses a gap-based approach for initial injectable selection.

A. If current HbA1c and target are both available:

  • Calculate gap = HbA1c − HbA1c target.

B. If current HbA1c is available but target is not provided:

  • Use a default HbA1c target of 7.0%.
  • Then calculate the gap against 7.0%.

C. If gap < 2% above target:

  • If BMI ≤ 30 kg/m² → start basal insulin.
  • If BMI > 30 kg/m² → start FRC.

D. If gap ≥ 2% above target:

  • Start FRC, regardless of BMI.

E. If current HbA1c is not available:

  • If BMI > 30 kg/m² → start FRC.
  • If BMI ≤ 30 kg/m² or BMI is not available → start basal insulin.

7) Optional comments outside the main routing tree

The algorithm may add a note that standalone GLP-1 RA could be discussed as an optional out-of-pocket consideration, but this does not affect the main decision tree.

This tool is intentionally reimbursement-oriented: standalone GLP-1 RA is not used as a routing branch in this version.

Disclaimer

This tool is a clinical decision-support prototype for internal evaluation. It does not replace clinical judgement, local guidance, institutional policies, or product labels. Always individualise therapy. Off-label situations, reimbursement restrictions, and local access conditions should be reviewed before implementation.
